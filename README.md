# SKAX Github Enterprise Cloud 사용 임시 가이드 

`2026년 사내 표준 깃허브 사용 정책 및 가이드는 준비 중에 있습니다.`

## 현재 임시 깃허브 신청링크는 아래와 같습니다.

[http://skaxgit.koreacentral.cloudapp.azure.com:8080/git-manager/](http://skaxgit.koreacentral.cloudapp.azure.com:8080/git-manager/)


* 위 링크는 임시용으로 사내 로컬망에서만 접속 가능합니다. 접속이 불가능한 분만 기존과 같이 기존 Forms 이용 바랍니다.
  * ``` Forms 링크는 VDI 내부 https://aihow.skax.co.kr 공지사항에 올라와 있습니다. ```
* 단, 계정을 신청하려면 다음 깃허브 프로파일 표준을 준수해야 합니다.
  * [사내용 깃허브 계정 표준](https://github.com/skax-internal-system/skax-github-guide/blob/main/ghec-profile-guide.pdf)
   
   
## 깃허브 계정 주의사항
- 신청서에 대해 관리자가 승인하면 깃허브에서 초청 메일이 발송됩니다.
- 깃허브 Enterprise 멤버십 초청 메일 수신후 7일 내에 메일인증을 완료해야 하며, 7일 경과시는 초청장이 자동 만료됩니다.(만료 후에는 재신청 필요)
- 신청한 조직/프로젝트 코드로 깃허브 Enterprise 사용료가 매달 이관 요청됩니다. (CL/AI 사용료 예산이 있어야 함)
- 깃허브 정책은 계속 업데이트 중이라 변경될 수 있으며
- 코파일럿은 신청자 계정이 깃허브 Enterprise 조인 완료된 후에 신청 가능합니다.
  (조인이 완료되지 않은 상태에서는 기능적으로 코파일럿 할당 불가)

## 깃허브 계정 삭제 관련
- 원칙적으로 깃허브 계정이 삭제되는 것이 아니라 (계정은 개인 것이라 그럴 권한이 없음), AX 표준저장소에서 멤버십 탈퇴처리됩니다.
- 멤버십 탈퇴는 다음의 경우에 발생합니다. (사전 통보 없이 멤버십 삭제)
 1. 프로필 표준을 준수하지 않는 경우
 2. 비용정산 대상 프로젝트 코드와 매핑되지 않는 경우
 3. 깃허브 Enterprise에 조인 후 30일 이상 활동이 없는 경우 
 4. Enterprise 멤버십 획득 이후에 Username을 변경한 경우 : 멤버십 관리를 위해 Username은 중요한 키가 되는데, Username 변경할 경우 관리자가 승인하지 않은 계정이 되어 강퇴됩니다.

## 깃허브 비용부과 프로세스
- 깃허브 사용료는 관리자가 계산하는 것이 아니며, 깃허브 Enterprise에서 Usage Record를 계산하여 일괄 청구합니다.
- 청구된 내역에 대해 신청 계정 정보의 프로젝트/조직 코드와 매핑하여 해당 조직으로 비용이관 처리됩니다. (담당 PM/팀장을 참조)
- 따라서, 프로젝트 코드가 매핑되지 않는 계정은 삭제될 수 있습니다.

## 깃허브 활동 로그 감시
깃허브 활동 로그 추적은 **소스 관리 및 보안 감사**의 목적이기 때문에 다음의 활동들만 인지됩니다.
- 소스 관련 작업 (push / pull / commit 등)
- workflow 실행
- 리파지토리 환경변수 작업
- 리파지토리 owner인 경우, 팀 멤버관리 작업 등
- Issue 할당 및 관련된 코드 작업

깃허브 활동 로그에서 **인지되지 않는 사용자 활동**은 다음과 같습니다.
- 웹 화면에서 단순 로그인
- Issue 생성 및 Comment 등록 

_따라서, 깃허브에서 소스 및 DevOps 작업에 참여하지 않고 이슈만 관리하는 경우 깃허브 활동 로그에서 추적할 수 없으므로 비활동 휴면 사용자로 간주될 수 있습니다._
